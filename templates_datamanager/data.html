Cohorts

A cohort is a named group of slides, matched by a simple name pattern (e.g. um* means "every slide whose folder starts with um" → cohort upmc). Editing patterns never moves or deletes any files — it only regroups slides. Changes take effect when you Apply changes.

Pick a folder of slides (a whole cohort, or any subset). Each slide is uploaded separately, then Apply changes to add them to the review set.

Expected folder structure — one subfolder per slide: 
my_cohort/                 ← the folder you pick
  um072/                   ← one slide (folder name = slide id)
    clean/   *.jpeg        ← tile images (or put images at the slide root)
    labels/  *.txt         ← one YOLO .txt per image (same basename)
  um073/
    clean/   *.jpeg
    labels/  *.txt
  …
Name slide folders with the cohort prefix (e.g. um* → cohort upmc), or add a matching pattern in Organize cohorts. Picking a single slide folder (one that directly contains labels/) uploads just that slide.
After uploading, go to Apply changes to add the new slides to the review set.
How big can this be? Uploads go over HTTPS through Cloudflare, which caps each request at 100 MB on the free plan — so a cohort is sent one slide at a time (a slide is typically ~50 MB). Any single slide over ~95 MB is skipped here; copy those to the server directly (rsync/scp), then Apply changes. Re-uploading an existing slide folder is rejected (it would re-map already-answered detections).
