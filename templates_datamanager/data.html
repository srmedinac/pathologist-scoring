Cohorts

A cohort is a named group of slides, matched by a simple name pattern (e.g. um* means "every slide whose folder starts with um" → cohort upmc). Editing patterns never moves or deletes any files — it only regroups slides. Changes take effect when you Apply changes.

Pick a folder of slides and give it a cohort name. Slides are stored under that cohort (patches_dir/<cohort>/<slide>/), so the same slide or image names can appear in different cohorts without clashing — nothing from another cohort is ever replaced. Each slide is uploaded separately; then Apply changes to add them to the review set.

Expected folder structure — the folder you pick is the cohort, with one subfolder per slide. It lands on the server as: 
patches_dir/
  <cohort>/                 ← the cohort name (from the folder you pick)
    um072/                  ← one slide
      clean/   *.jpeg       ← tile images (or images at the slide root)
      labels/  *.txt        ← one YOLO .txt per image (same basename)
    um073/
      clean/   *.jpeg
      labels/  *.txt
After uploading, make sure a cohort in Organize cohorts has a pattern matching <cohort> (e.g. the exact name) so raters can be assigned to it.
After uploading, go to Apply changes to add the new slides to the review set.
How big can this be? Uploads go over HTTPS through Cloudflare, which caps each request at 100 MB on the free plan — so a cohort is sent one slide at a time (a slide is typically ~50 MB). Any single slide over ~95 MB is skipped here; copy those to the server directly (rsync/scp), then Apply changes. The only thing rejected is re-uploading the same slide into the same cohort — that would re-map already-answered detections; the same name in a different cohort is fine.
